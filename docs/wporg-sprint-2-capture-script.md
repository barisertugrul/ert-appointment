## A) Hazırlık (4 dk)

1. Admin paneline giriş yap (`Appointment Booking`).
2. Demo içerik görünürlüğünü kontrol et:
   - En az 3-5 randevu satırı olsun.
   - Farklı durumlar (ör. confirmed/pending) görünür olsun.
3. Görünüm standardını sabitle:
   - Tarayıcı zoom: `%100`
   - Tek sekmede temiz arayüz (gereksiz panel/eklenti kapalı)

## B) screenshot-3.png (13 dk)

Hedef:
- Admin dashboard / appointments overview ekranını operasyonel ve temiz göstermek.

Adımlar:
1. `Appointments` veya dashboard overview ekranını aç.
2. Tarih filtresi veya liste görünümü aktifse, en anlaşılır görünümü seç.
3. Aşağıdaki öğeler görünür olsun:
   - Randevu listesi/overview tablosu
   - Temel sütunlar (tarih/saat, müşteri, provider/hizmet, durum)
   - Mümkünse kısa aksiyon alanı (detay/düzenle)
4. Kişisel verileri anonimleştir (isim/e-posta/telefon demo olmalı).
5. Ekranı ortalayarak görüntü al.
6. Dosyayı kaydet:
   - `screenshot-3.png`

Hızlı kalite kontrol:
- [ ] Liste net okunuyor
- [ ] Durumlar görünür
- [ ] Gerçek müşteri verisi yok

## C) screenshot-4.png (10 dk)

Hedef:
- Forms page üzerinde drag-drop builder ve booking button text özelleştirmesini göstermek.

Adımlar:
1. `Forms` sayfasına geç.
2. Form düzenleme panelini aç.
3. Aşağıdaki alanlar aynı karede mümkün olduğunca görünsün:
   - Form field listesi (drag-drop hissi veren bölüm)
   - Alan düzenleme/ayar paneli
   - `Booking Button Text` (veya karşılığı) ayarı
4. Buton metni örnek değerle görünür olsun:
   - Örn: `Book Appointment` veya `Randevu Al`
5. Ekranı sade ve okunur kadrajla yakala.
6. Dosyayı kaydet:
   - `screenshot-4.png`

Hızlı kalite kontrol:
- [ ] Form builder net görünüyor
- [ ] Booking button text özelliği görünür
- [ ] UI kırığı veya taşma yok
